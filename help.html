shield Content Requirements

What you can and cannot include in your content

1.1 Appropriate Content

Your content must not contain:

  • Malicious code, viruses, or any harmful software
  • Offensive, discriminatory, or hateful content
  • Sexually explicit or pornographic material
  • Content that promotes violence, illegal activities, or self-harm
  • Copyrighted material without proper authorization
  • Spam or misleading information

1.2 Privacy & Security

  • Do not collect user data without explicit consent
  • Do not include tracking scripts or analytics without disclosure
  • Protect any user input or stored data appropriately
  • Do not attempt to access or modify other users' data

code Technical Requirements

Size limits, metadata, and functionality standards

2.1 File Size Limits

ℹ️ Size Limit: Your app's HTML content must be less than 2500 characters or 1MB when encoded as Base64.

  • Optimize your code to stay within the size limit
  • Use external CDNs for large libraries when possible
  • Minify your HTML, CSS, and JavaScript

2.2 Metadata Standards

Your content must include:

  • Name: A clear, descriptive name (no offensive language)
  • Description: An accurate description of what your content does
  • Author: Your name or username
  • Version: A valid version number (e.g., 1.0.0)
  • Tags: Relevant tags to help users find your content

2.3 Functionality

  • Your content must be functional and not intentionally broken
  • Test your content before publishing
  • Ensure compatibility with modern web browsers
  • Handle errors gracefully

upload Publishing Process

How to publish and update your apps

3.1 From App Creator

✓ Easy Publishing: Use the "Export for Publishing" button in the App Creator, then choose "Community" when prompted.

  1. Create your app in the App Creator
  2. Click "Export for Publishing"
  3. Choose "Publish to Community" (no approval needed)
  4. Enter your author name
  5. Confirm you've read and agree to these guidelines
  6. Enter the version number
  7. Add relevant tags for discoverability
  8. Acknowledge that the app is your original work
  9. Submit your app

3.2 Updates

  • To update an existing app, use the same App ID
  • Increment the version number (e.g., 1.0.0 → 1.0.1)
  • Users will be notified of available updates

brush Generic Styling

How to build apps using the platform's generic styling baseline.

Why use Generic Styling

Generic Styling gives your app a cohesive look and reduces duplication of styles across apps. It also helps keep individual app bundles smaller by sharing common CSS.

Practical steps

  • Link or import styles/generic-app-style.css where possible.
  • Override only the variables or utility classes you need; avoid copying the entire stylesheet into your app.
  • Scope custom CSS using prefixed class names (e.g., .myapp-*) to avoid conflicts.
  • Use CSS custom properties for theming so you can change colors without large duplications.

cloud External APIs & CDNs

Guidance for using remote services and CDN libraries safely and efficiently.

Using fetch() and remote APIs

Use the browser's fetch() API to retrieve data. Keep responses small and cache where appropriate.

  • Use HTTP caching headers on your API to reduce repeated downloads.
  • Limit request payloads and pagination results to what you need.
  • Implement graceful fallbacks for when the API is unavailable.

Including CDN libraries

  • Load only the parts of a library you need (CDNs often provide modular bundles).
  • Use integrity attributes (integrity) when available to protect against tampering.
  • Prefer async or defer script loading to avoid blocking rendering.

compress Size Optimization

Concrete techniques to reduce final app size.

Techniques

  • Minify assets, remove comments, and strip development-only code.
  • Use tree-shaking where possible to drop unused JS.
  • Prefer srcset and responsive images to serve smaller images on mobile.
  • Delay loading of large resources; only inline truly critical CSS/JS.

code App Creator Overview

Learn about the App Creator tool and its features

What is App Creator?

App Creator is a powerful web-based tool that allows you to create, edit, and publish custom web applications directly in your browser. It features a Monaco code editor (the same editor used in VS Code) and provides an intuitive interface for managing your apps.

Key Features

  • Monaco Editor: Professional code editing with syntax highlighting, auto-completion, and error detection
  • Asset Management: Upload and manage images and other assets for your apps
  • Live Preview: See your app changes in real-time
  • Import/Export: Save your apps as XML files for backup or sharing
  • Community Publishing: Share your apps with the community
  • Keyboard Shortcuts: Use Ctrl+S to save quickly

add Creating New Apps

How to start building your first app

Getting Started

To create a new app:

  1. Open the App Creator from the main menu
  2. Click the "Create New App" button
  3. Enter a name for your app (required)
  4. Click "Create App" to start editing

App Properties

After creating an app, you can configure:

  • Name: The display name of your app
  • Description: A brief description of what your app does
  • Icon: Upload or paste an icon for your app
  • Options: Light mode compatibility, open source status, and style templates

edit Editing Your Code

Using the code editor effectively

The Code Editor

The App Creator uses Monaco Editor, providing:

  • Syntax highlighting for HTML, CSS, and JavaScript
  • Auto-completion and IntelliSense
  • Error detection and suggestions
  • Find and replace functionality
  • Multiple cursor support

Menu Buttons

The code section has several toolbar buttons:

  • Save: Save your current code changes
  • Exit: Return to the home view
  • Paste Image: Insert images from your clipboard
  • Preview: Open your app in a new window

Keyboard Shortcuts

  • Ctrl+S: Save your app
  • Ctrl+Z: Undo
  • Ctrl+Y: Redo
  • Ctrl+F: Find

image Managing Assets

How to add and manage images and files

Asset Management

Assets are images and files used in your apps. The Assets tab shows all assets for your current app.

Adding Assets

You can add assets in several ways:

  • Upload: Click "Upload Asset" to select files from your computer
  • Paste: Copy images and paste them directly into the code editor
  • From Clipboard: Use the "Paste Image" button in the code editor

Using Assets in Code

When you paste or upload an asset, it gets an ID. Use this ID in your HTML:

<img src="asset-id" alt="My Image">

Managing Assets

For each asset, you can:

  • View: Open the asset in a new tab
  • Replace: Upload a new file to replace the current one
  • Delete: Remove the asset from your app

publish Publishing Your Apps

Share your apps with others

Publishing Options

You can publish your apps in different ways:

Community Publishing

Share your app with the community:

  1. Ensure your app is saved
  2. Click "Publish App" in the Properties tab
  3. Select "Publish to Community"
  4. Your app will appear in the App Store for others to use

Featured Publishing

To get your app featured:

  1. Export your app as XML
  2. Email the XML file to shawnyxx@ecxogames.ca
  3. Include your app name, description, and open source status

Manual Instructions

The "Manual Instructions" option provides detailed email templates and export guides.
